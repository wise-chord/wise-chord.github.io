# Nazu 語言與其可執行子集系統化概述

Nazu 是一門結合了邏輯語（Loglang）與函數式編程語義的人工語言。其語意設計的核心在於「句法即邏輯」，將複雜的自然語言關係簡化為「單一論元加上關係標記片語（CP）」的對稱結構。在可執行子集中，這一模型被映射為基於原型、代數資料型別（ADT）與強型別約束的程式語言。

---

## 一、 語音與文字系統 (Phonology & Orthography)

Nazu 採用拉丁字母轉寫，其國際音標（IPA）發音對應關係如下：

### 1. 子音 (Consonants)
*   **塞音**：`p` [pʰ], `b` [b], `t` [tʰ], `d` [d], `k` [kʰ], `g` [g], `h` [ʔ]（聲門塞音）
*   **鼻音**：`m` [m], `n` [n], `q` [ŋ]
*   **擦音與塞擦音**：`f` [f], `v` [v], `s` [s], `z` [z], `c` [ʃ], `j` [ʒ], `x` [h]
*   **流音**：`l` [l], `r` [r/ɹ/ɻ]（齒齦顫音、近音或捲舌近音）

### 2. 母音 (Vowels)
*   `a` [a/ä/ɑ], `i` [i/ɪ], `u` [u/ʊ], `e` [e/ɛ], `o` [o/ɔ], `y` [ə]

## 二、 語言學核心：概念 (N) 與限定 (D)

Nazu 不區分傳統的名詞、動詞、形容詞，所有實詞均為表達狀態或關係的「概念（$N$）」。

### 1. 概念（N）與默認論元（Default Argument）
每個 $N$ 隱式帶有一個核心語意對象，稱為**默認論元**：
*   若 $N$ 表達動作/狀態（如 `vid`：是看見），其默認論元為**被看之物**。
*   若 $N$ 表達屬性（如 `sa`：是大、`mio`：是藍色），其默認論元為**比較的標準**。
    *   `l raq mio`：意為「該處所與標準一樣藍（呈現等同的藍色狀態）」。

### 2. 限定詞（D）與限定片語（DP）
$N$ 無法直接充當句法論元，必須結合限定詞 $D$ 投影為 $DP$。
*   **附屬限定詞**：`l`（定指/通用）, `ex`（存在一個/幾個）, `al`（全稱/所有）, `tu`（不定指/某個）, `su`（特指）。
*   **疑問限定詞**：`qe`（修飾 $N$，如 `qe raq` 哪個地方）。
*   **獨立限定詞**：自身即是 $DP$，包含 `mi`（我）、`tu`（你）、`qi`（誰/什麼，不接 $N$）。
*   **深層縮合邏輯**：獨立限定詞（如 `mi`、`tu`、`qi`）本質上是定指 `l` 與某個隱性概念 $N$（如 $[N_{ego}]$ 說話者）的縮合。因此，它們深層仍保留一個 $N$，並能正常掛載關係片語（CP）進行修飾。

## 三、 句法骨架與封閉詞系統 (Syntax & Closures)

Nazu 採用主題優先（Topic-Prominent）句法。一個子句只能有一個主題，位於句首。

### 1. 隱式 C（語法糖）與價位判定
在「主題 + N + 默認論元」的骨架中，主題的角色由論元數量決定：
*   **一價狀態（無額外論元）**：主題自動成為該 $N$ 的默認論元。
    *   `qe raq mio` $\rightarrow$ 以哪一個地方為標準的藍色（即：哪個地方的藍色）。
*   **二價狀態（存在默認論元）**：主題自動成為施事（Agent）、感事（Experiencer）或不可抗力。
    *   `mi vid l raq` $\rightarrow$ `l raq` 佔用默認論元位置，主題 `mi` 自動成為看見的施事（我看見這個地方）。

### 2. 關係標記（C）與屬性片語（CP）
非默認的論元關係必須由關係標記 $C$ 導出 $CP$：
*   `in`：處於……時空位置。
*   `i`：具有……性質。
*   `qa`：標記施事/感事（用於被動或施事降格）。
*   `ur`：使用……工具。
*   `de`：屬於……。
*   `av`：擁有……。
*   `r`：內置屬性標記。其 $C'P$ 的中心語為空，用於將特定屬性封閉（例如 `rio r` 表示「顏色是……的屬性」）。

### 3. 完整封閉詞系統
為了消除句意嵌套時的範疇歧義，Nazu 設計了嚴格的封閉詞標記：
*   `f`：關閉關係片語（CP）。
*   `s`：關閉限定片語（DP），縮寫為逗號 `,`。
*   `r`：關閉內置屬性片語（C'P）。
*   `t`：關閉主題化移動片語（TP），標記主題從內部移出的語跡終點。
*   `p` = 關閉概念片語（NP）。
*   `;`：多功能封閉詞，底層對應 `f/t/p` 的關閉。若 `;` 的後方緊鄰 $DP$ 時，會自動在兩者之間插入性質標記 `i`。

#### 命名與指代系統（lA 序列）
Nazu 無原生的第三人稱代名詞，需使用連綴詞 `a` 與 `lA` 系列（`la`、`li`、`lu`、`le`、`lo`）臨時定義：
*   **語法結構**：`[DP] s a la`
    *   若要指代「一個很小的處所」（`l raq m i l sie-sa`），必須精確封閉為：
        `l raq m i l sie-sa f s a la`（由 `f` 封閉 CP，再由 `s` 封閉整個 DP，最後以 `a` 命名為 `la`）。

### 4. 繫詞 es 與形式變量 m
*   **es (斷言)**：表示兩個實體被斷言為相同（$A \equiv B$）。如 `mi es l vid`（我就是「看」這個行為）。
*   **m (形式 N)**：本身無語義，不帶默認論元，專用於掛載顯式 $C$ 以陳述偶發狀態。
    *   錯誤：`*tu es in l raq`（你等同於「在處所中」的時空）。
    *   正確：`tu m in l raq`（你處於「在處所中」的狀態）。

## 四、 邏輯、時態與複雜語意

### 1. 邏輯關係 $N$
邏輯關係以主導的 $N$ 表示（結構為 `A 邏輯N B`）：
*   `si`（蘊含/如果）：`A si B`
*   `tuo`（導致/因為）：`A tuo B`
*   `e`（合取/順承/轉折）：`A e B`
*   `o`（析取）：`A o B`
*   `keo`（強轉折/雖然）：`A keo B`
    *   *註*：並列 CP 時，必須用封閉詞 `f` 隔開，即 `CP f CP`，避免層級吞噬。

### 2. 時態、頻率與主題約束
*   **時間 CP**：`in l pu`（過去）、`in l sian`（現在）、`in l fiu`（未來）。
*   **頻率 N**：`pulguo`（總是）、`nuiguo`（有時）、`nyguo`（從不）。
*   **單一主題約束**：若將時間或頻率（如 `in l pu`）移至句首充當主題，原句子的主體（施事）必須降格，使用 `qa` 標記並移至句尾。
    *   `in l pu pulguo l vid la f qa mi`（在過去，它總是為我所見）。

### 3. 情態與方式
*   **情態 N**：`vol`（想要）、`pot`（能夠）、`deb`（必須）。
    *   `mi vol l vid tu`（我想要看見你）。
*   **方式副詞（Scheme B）**：`qa mi suai toir [動作DP]`（我快速地進行[某動作]，toir（動作為））。

### 4. 比較級系統
利用形容詞性 $N$（如 `sa`，大）的默認論元為「標準」這一特性構建：
*   **同等比較**：`A m i l tuq l sa de B`（A 具有與 B 等大的性質）。
*   **比較級**：`A m i l sa l sa de B`（A 具有以 B 為標準而大的性質，即 A 比 B 大）。
*   **最高級**：`A m i l sa l sa de al B`（A 具有在所有 B 範圍內為標準而大的性質）。

## 五、 特殊句型與數值

1.  **疑問句**：
    *   **是非問**：焦點後加 `on`（`tu vid la s on` 你看見它嗎？）。
    *   **選擇/特指問**：使用 `qe` 或 `qi`，可留置原位或置於句首主題位置。
2.  **被動語態**：不改變動詞，直接用 `qa` 標記施事，未標記的主題自動充當受事（`mi vid qa tu` 我被你看見）。
3.  **祈使句**：省略句首主題，直接以 $NP$（核心 $N$ + 默認論元）開頭（`vid l raq` 看這地方！）。
4.  **數字系統**：數字可作為 $D$ 或 $N$，原生最高支援十六進位，一般用十進位。
    *   數位：`na` (0), `pi` (1), `gu` (2), `fe` (3), `so` (4), `da` (5), `ji` (6), `ku` (7), `me` (8), `xo` (9), `ta` (10/A), `ci` (11/B), `ru` (12/C), `be` (13/D), `qo` (14/E), `za` (15/F)。
    *   高位數表達，如十六進位的 `pi-na` 代表 10（即十進位的 16）。
    *   範例：`fe raq`（三個地方）。

## 六、 可執行子集語意系統

我們將 Nazu 的語言學句法映射為計算機的型別與運算系統。

### 1. 符號與運算子映射表

| 符號 | Nazu 映射 | 語意 |
| :--- | :--- | :--- |
| `=` | `es` | 賦值 / 絕對同等 |
| `:` | `u` | 型別約束 |
| `,` | `s` | 參數分隔符（DP 封閉詞） |
| `;` | `f/t/p` | 語句結束。若後方接 DP 則自動補 `i` |
| `\|` | `siar` | 函數輸入參數列表宣告（需求） |
| `?` | `si` | 條件分支（如果 / 蘊含） |
| `!` | `uo` | 分支否則（Else） |
| `+` | `av` | 擁有 / 原型擴充 |
| `^` | `xo` | 互斥析取（多個中選一個） / 和型別變體分隔 |
| `.` | `de` | 屬性訪問（順序需要顛倒） |

---

### 2. 資料結構與型別定義

#### 結構體（原型擴充與列多型）
結構體是基於空原型 `nil` 通過 `+`（`av`）算子不斷擴充屬性所構造的類型。

```Nazu
// Vector3 原型定義
Vector3 = nil
  + x: f32;, y: f32;, z: f32;;
;
```
*   **實例化與屬性覆寫**：利用 `@`（內部屬性，@name "a"; 在內部會被解析成 name_N r "a" f）覆寫原型中的虛擬欄位，並用分號分隔：
```Nazu
p1 = Vector3
  x r 1.0;
  y r 2.0;
  z r 3.0;
;
```

#### 枚舉（和型別 / Tagged Union）
利用互斥析取詞 `^` 連接不同的原型標籤，實現安全的分支型別定義：
```Nazu
Number = 
    I + i32;,
  ^ F + f64;
;
```

### 3. 函數與免括號邊界解析

在 Nazu 可執行子集中，函數並非指令流，而是**受前置條件限制並在滿足時求值的值**。其參數邊界不強制依賴括號 `()`，而是透過分號 `;` 與逗號 `,`（即 `s` 封閉詞）進行解析。

#### 階乘函數範例
```Nazu
fact = 1;
  | 0;
;
fact = res: i32;
  | n: i32;;
  l mul n, fact (minus n, 1);
;
```

#### 免括號參數邊界解析範例
在呼叫 `eq (mod n, 2), 0` 時，若不使用括號，必須寫為：
```Nazu
eq mod n, 2;, 0
```
*   **編譯器解析過程**：
    1.  遇到 `mod`，探測其參數。第一個參數為 `n`，隨後遇到 `,`（即 `s`，DP 封閉符），第二個參數為 `2`。
    2.  遇到 `;,`（即 `p` NP 封閉符與 `s` DP 封閉符），將 `mod n, 2` 打包封閉為一個獨立的 $DP$ 表達式（參數 1）。
    3.  回到 `eq` 層級，接受下一個參數 `0`。
    4.  整體被解析為以 `mod n, 2` 的求值結果與 `0` 作為參數傳給 `eq`。

#### 底層 AST 無語法糖展開
上述免括號表達式在編譯時，會被精確地解糖還原為底層的純 Nazu 句法結構：
`l eq l mod l n s l 2 p s l 0`

這種嚴格的代數對稱性，使得 Nazu 的代碼無需依賴複雜的語法解析器，即可直接映射為語意清晰、無二義性的語義網結構。
